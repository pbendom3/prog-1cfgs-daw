Diagrama UML de una clase abstracta

Si te fijas, en PlantUML siempre se ha generado un símbolo C al lado de las clases que hemos ido creando:

Cuando una de estas clases es abstracta, ese símbolo C se modificar por A:

@startuml

    abstract class Persona {
        - String nombre
        - int edad
        + Persona(String nombre, int edad)
        + mostrarInfo()
        + accederEvento()  // Método abstracto
    }

    class Artista {
        - String generoMusical
        + Artista(String nombre, int edad, String generoMusical)
        + mostrarInfo()
        + accederEvento()
    }

    class Asistente {
        - String entrada
        + Asistente(String nombre, int edad, String entrada)
        + mostrarInfo()
        + accederEvento()
    }

    class Organizador {
        - String rol
        + Organizador(String nombre, int edad, String rol)
        + mostrarInfo()
        + accederEvento()
    }

    Persona <|-- Artista
    Persona <|-- Asistente
    Persona <|-- Organizador

@enduml
