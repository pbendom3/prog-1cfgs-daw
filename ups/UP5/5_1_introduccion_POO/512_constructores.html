5.1.2. Constructores

Cuando se construye un objeto es necesario dar un valor inicial a sus atributos. Es por ello que existe un método especial en cada clase, llamado constructor, que es ejecutado de forma automática cada vez que es instanciada una clase (realizamos un new). 

El constructor se llama igual que la clase y no devuelve nunca ningún valor a través del return. La invocación al método constructor devolverá una instancia de dicho objeto.

public class Persona {
    String nombre; //atributo nombre
    String dni; //atributo dni
    int edad; //atributo edad

    //constructor de la clase Persona
    public Persona(String nombre, String dni, int edad) {
        this.nombre = nombre;
        this.dni = dni;
        this.edad = edad;
    }

}

Y desde el main:

public class Main {
    public static void main(String[] args) {
        //creación de dos objetos Persona
        Persona persona1 = new Persona("Juan Pérez", "12345678A", 30);
        Persona persona2 = new Persona("Ana García", "87654321B", 25);

        System.out.println(persona1.nombre);
        System.out.println(persona2.nombre);
    }
}

Sobrecarga de constructores

Una clase puede tener varios constructores, que se diferencia por el tipo y su número de parámetros. Igual que pasaba con los métodos, pueden tener la misma cantidad de parámetros, pero como mínimo, uno de ellos tiene que ser de un tipo distinto. O directamente tener parámetros distintos, como en el siguiente caso:

public class Persona {
    //atributos con visibilidad protegida
    String nombre;
    String dni;
    int edad;

    //constructor parametrizado de la clase Persona
    public Persona(String nombre, String dni, int edad) {
        this.nombre = nombre;
        this.dni = dni;
        this.edad = edad;
    }

    //constructor parametrizado pero sin el atributo edad
    public Persona(String nombre, String dni) {
        this.nombre = nombre;
        this.dni = dni;
    }

}

Y desde el main:

public class Main {
    public static void main(String[] args) {
        //creación de dos objetos Persona
        Persona persona1 = new Persona("Juan Pérez", "12345678A", 30);
        Persona persona2 = new Persona("Ana García", "87654321B", 25);
        Persona persona_sin_edad = new Persona("Carlos Díaz", "547896514A");

        System.out.println(persona1.nombre);
        System.out.println(persona2.nombre);
        System.out.println(persona_sin_edad.nombre + " " + persona_sin_edad.edad);
    }
}

Además, en Java existen los llamados constructores por defecto (sin parámetros) que inicializan un objeto a:

  • En caso de ser tipos primitivos, a su valor por defecto.
  • Strings y referencias a objetos a null.
public class Persona {
    //atributos con visibilidad protegida
    String nombre;
    String dni;
    int edad;

    //constructor parametrizado de la clase Persona
    public Persona(String nombre, String dni, int edad) {
        this.nombre = nombre;
        this.dni = dni;
        this.edad = edad;
    }

    //constructor por defecto de la clase Persona
    public Persona() {

    }

}

Este constructor por defecto es útil si necesitamos crear objetos de Persona que serán inicializados más tarde, o si los atributos secundarios no son necesarios al momento de la creación del objeto.

public class Main {
    public static void main(String[] args) {
        //creación de dos objetos Persona
        Persona persona1 = new Persona("Juan Pérez", "12345678A", 30);
        Persona persona2 = new Persona("Ana García", "87654321B", 25);
        Persona persona3 = new Persona();

        System.out.println(persona1.nombre);
        System.out.println(persona2.nombre);
        System.out.println(persona3.nombre);
    }
}
